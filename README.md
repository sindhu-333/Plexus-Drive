## 🚀 Quick Start Guide

### **Step 1: Clone Repository**
```bash
# Clone the repository
git clone https://github.com/sindhu-333/Plexus-Drive.git

# Navigate to project directory
cd plexus-drive
```

### **Step 2: Install Dependencies**

#### Backend Setup
```bash
cd backend
npm install --production
```

#### Frontend Setup
```bash
cd frontend
npm install --production
```

### **Step 3: Database Configuration**

#### Create PostgreSQL Database
```sql
-- Connect to PostgreSQL
psql -U postgres

-- Create database
CREATE DATABASE plexus_db;

-- Grant privileges
GRANT ALL PRIVILEGES ON DATABASE plexus_db TO postgres;
```

#### Initialize Database Schema
```bash
psql -U postgres -d plexus_db -f database.sql
```

### **Step 4: Environment Configuration**

#### Backend Environment Variables
Create `backend/.env`:
```env
# 🌐 Server Configuration
PORT=5000
NODE_ENV=development

# 🐘 Database Configuration
DB_HOST=localhost
DB_PORT=5432
DB_NAME=plexus_db
DB_USER=postgres
DB_PASSWORD=your_secure_password_here

# 🔐 JWT Authentication
JWT_SECRET=generate_random_secure_key_here
JWT_EXPIRES_IN=2h

# ☁️ Dropbox Integration
DROPBOX_REFRESH_TOKEN=your_dropbox_refresh_token
DROPBOX_APP_KEY=your_dropbox_app_key
DROPBOX_APP_SECRET=your_dropbox_app_secret

# 📧 Email Configuration (Gmail SMTP)
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_SECURE=false
EMAIL_USER=your_email@gmail.com
EMAIL_PASS=your_gmail_app_password
EMAIL_FROM=your_email@gmail.com

# 🌐 Frontend URLs (Auto-detected)
# EMAIL_FRONTEND_URL=http://localhost:3000
# BACKEND_URL=http://localhost:5000
```

#### Frontend Environment Variables
Create `frontend/.env`:
```env
# 🔗 API Configuration
VITE_API_URL=http://localhost:5000/api
VITE_BACKEND_URL=http://localhost:5000
```

### **Step 5: Dropbox API Setup**

1. 🌐 Visit [Dropbox App Console](https://www.dropbox.com/developers/apps)
2. ➕ Create new app → Choose "Scoped access" → Select "Full Dropbox"
3. 🔑 Navigate to "Settings" → Copy **App Key** and **App Secret**
4. 🎯 Set **Redirect URI**: `http://localhost:5000`
5. 🔄 Generate refresh token:
   ```bash
   cd backend
   node get_refresh_token.js
   ```
6. 📋 Copy the refresh token to your `.env` file

### **Step 6: Gmail SMTP Configuration**

1. 🔐 Enable **2-Factor Authentication** on your Google Account
2. 🔑 Go to [App Passwords](https://myaccount.google.com/apppasswords)
3. 📧 Generate password for "Mail" → Copy 16-digit password
4. 📝 Add to `backend/.env` as `EMAIL_PASS`

### **Step 7: Launch Application**

#### Terminal 1 - Backend Server
```bash
cd backend
npm start
```

#### Terminal 2 - Background Worker (Optional)
```bash
cd backend
node worker.js
```

#### Terminal 3 - Frontend Development Server
```bash
cd frontend
npm run dev
```

### **Step 8: Access Application**

| Service | URL | Description |
|---------|-----|-------------|
| 🌐 **Frontend** | http://localhost:3000 | User interface |
| 🔧 **Backend API** | http://localhost:5000 | REST API server |
| 📊 **Health Check** | http://localhost:5000/api/auth/health | Server status |

### **Network Access**

Access from other devices on the same network:
```
Frontend: http://[YOUR_LOCAL_IP]:3000
Backend:  http://[YOUR_LOCAL_IP]:5000
```

> 💡 **Tip**: The system automatically detects your network IP and adapts all URLs accordingly!

## 🗄️ Database Architecture

### **Entity Relationship Overview**

```
┌─────────────┐      ┌──────────────┐      ┌─────────────┐
│   Users     │──────│    Files     │──────│   Folders   │
└─────────────┘      └──────────────┘      └─────────────┘
      │                     │                      │
      │                     │                      │
      ▼                     ▼                      ▼
┌─────────────┐      ┌──────────────┐      ┌─────────────┐
│Notifications│      │  File Shares │      │  File Tags  │
└─────────────┘      └──────────────┘      └─────────────┘
                            │
                            ▼
                     ┌──────────────┐
                     │ Access Logs  │
                     └──────────────┘
```

### **Core Tables**

#### 👤 **users**
Stores user account information and authentication data
- `id` (PK) - Unique user identifier
- `email` - User email address (unique)
- `password` - Bcrypt hashed password
- `name` - User display name
- `email_verified` - Email verification status
- `profile_picture` - Avatar URL
- `theme` - UI theme preference
- `created_at` / `updated_at` - Timestamps

#### 📁 **files**
Manages file metadata and storage references
- `id` (PK) - Unique file identifier
- `user_id` (FK) - Owner reference
- `folder_id` (FK) - Parent folder
- `filename` - Original file name
- `filesize` - Size in bytes
- `mimetype` - Content type
- `dropbox_path` - Cloud storage path
- `status` - Processing status
- `created_at` / `updated_at` - Timestamps

#### 📂 **folders**
Hierarchical folder structure
- `id` (PK) - Unique folder identifier
- `user_id` (FK) - Owner reference
- `parent_id` (FK) - Parent folder (self-reference)
- `name` - Folder name
- `path` - Full folder path
- `created_at` / `updated_at` - Timestamps

#### 🔗 **file_shares**
File sharing configurations and permissions
- `id` (PK) - Unique share identifier
- `file_id` (FK) - Shared file reference
- `shared_by` (FK) - User who created share
- `share_token` - Unique access token
- `share_type` - Public/private
- `access_level` - View/download/edit
- `password_hash` - Optional password
- `expires_at` - Expiration timestamp
- `view_count` / `download_count` - Analytics
- `created_at` / `updated_at` - Timestamps

#### 🏷️ **file_tags**
AI-generated tags and classifications
- `id` (PK) - Tag identifier
- `file_id` (FK) - Tagged file
- `tag` - Tag name/keyword
- `confidence` - AI confidence score
- `tag_type` - Category classification
- `created_at` - Timestamp

#### 📊 **share_access_logs**
Audit trail for share access
- `id` (PK) - Log entry identifier
- `share_id` (FK) - Share reference
- `ip_address` - Accessor IP
- `user_agent` - Browser/client info
- `action` - View/download action
- `accessed_at` - Access timestamp

#### 🔔 **notifications**
User notification system
- `id` (PK) - Notification identifier
- `user_id` (FK) - Recipient user
- `title` - Notification title
- `message` - Notification content
- `type` - Success/warning/error/info
- `read` - Read status
- `created_at` - Timestamp

#### 🧠 **ai_file_analysis**
AI processing results storage
- `id` (PK) - Analysis identifier
- `file_id` (FK) - Analyzed file
- `analysis_type` - OCR/classification/etc
- `result` - JSON analysis data
- `confidence` - Confidence score
- `processed_at` - Processing timestamp

### **Indexes & Performance**

```sql
-- Performance indexes
CREATE INDEX idx_files_user_folder ON files(user_id, folder_id);
CREATE INDEX idx_shares_token ON file_shares(share_token);
CREATE INDEX idx_tags_file ON file_tags(file_id);
CREATE INDEX idx_notifications_user_read ON notifications(user_id, read);
```

### **Key Relationships**

- One user has many files, folders, shares, notifications
- One file belongs to one user and one folder
- One file has many shares and tags
- One share has many access logs
- Folders support self-referential hierarchy (parent_id)
